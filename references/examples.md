# Anti-Backrooms Design Examples

Dense examples for applying the skill across many artifact types.

## Quick Pattern

Each example follows:

- **Backrooms symptom**
- **AI failure**
- **Better move**

## UI / UX

### 1. Familiar surface, wrong logic

- **Backrooms symptom:** A room looks normal until you realize there is no reason it would exist in that shape.
- **AI failure:** A homepage includes hero, features, testimonials, stats, FAQ, and CTA because those are common website pieces, not because the product needs that exact structure.
- **Better move:** Start from the product's actual conversion path. Add only the sections that support it.

### 2. Impossible adjacency

- **Backrooms symptom:** A hallway opens into a space that should not connect to it.
- **AI failure:** Account settings sit beside onboarding guidance or destructive controls appear next to light, casual actions.
- **Better move:** Group by task gravity and user expectation, not by generic card symmetry.

### 3. Wrong scale

- **Backrooms symptom:** The room is technically usable but feels subtly too large or too small.
- **AI failure:** Buttons are visually balanced on desktop but too small for touch, or a mobile UI uses desktop density.
- **Better move:** Size for actual interaction, not for screenshot aesthetics.

### 4. Endless hum-buzz

- **Backrooms symptom:** Constant fluorescent noise.
- **AI failure:** Helper text, badges, labels, tooltips, and status lines everywhere.
- **Better move:** Reduce ambient UI noise so the primary action stands out.

### 5. Repetition without purpose

- **Backrooms symptom:** Corridor after corridor after corridor.
- **AI failure:** Eight identical feature cards, six identical stats, or a dashboard of repeated panels that all say nearly the same thing.
- **Better move:** Merge, prioritize, or differentiate. Repetition must teach a pattern, not reveal laziness.

### 6. Meta leakage

- **Backrooms symptom:** Reality has absorbed pieces of somewhere else.
- **AI failure:** A production site literally shows `Assignment 6`, `demo`, `draft`, `placeholder`, `internal`, or implementation-era naming.
- **Better move:** Strip process labels and replace them with user-facing language.

## Slides / Decks

### 7. False ending

- **Backrooms symptom:** You think you reached an exit, then the hall keeps going.
- **AI failure:** A deck lands its core story, then continues into more main-content slides instead of clearly switching to appendix or Q&A.
- **Better move:** Decide where the deck resolves. After that point, only appendix logic should remain.

### 8. Unreadable but technically present

- **Backrooms symptom:** Details exist, but the space refuses real human use.
- **AI failure:** Slide text is small enough that it only works from the laptop screen.
- **Better move:** Validate at room scale. Presentation text must survive projection and distance.

### 9. Appendix infection

- **Backrooms symptom:** Rooms start blending into one another.
- **AI failure:** Competitive analysis, financials, and detailed diagrams clutter the main 90-second story.
- **Better move:** Keep the main story clean. Move reference content into appendix slides.

### 10. Decorative density masking weak logic

- **Backrooms symptom:** Texture without orientation.
- **AI failure:** Lots of gradients, pills, cards, and shadows, but no clear story progression.
- **Better move:** Fix narrative order first, then style it.

### 11. Wrong beat per slide

- **Backrooms symptom:** The corridor doubles back emotionally even when it keeps moving visually.
- **AI failure:** One slide tries to carry problem, solution, market, and validation at the same time.
- **Better move:** One beat, one slide. If the beat is crowded, split or cut.

### 12. Premature detail

- **Backrooms symptom:** The room contains oddly specific objects before you even understand the space.
- **AI failure:** A slide shows implementation detail before the audience understands the core problem.
- **Better move:** Orient first. Detail later.

## Diagrams / Visualizations

### 13. Locally correct, globally unreadable

- **Backrooms symptom:** Paths exist, but navigation confidence does not.
- **AI failure:** A flow diagram has correct nodes, but readers cannot tell where to start or how to traverse it.
- **Better move:** Add explicit reading order, hierarchy, and directional bias.

### 14. Impossible proportions

- **Backrooms symptom:** Something is larger on the inside than the outside.
- **AI failure:** A tiny concept gets a giant box while critical infrastructure is compressed into a footnote.
- **Better move:** Match visual weight to conceptual importance.

### 15. Generic system diagram soup

- **Backrooms symptom:** Similar rooms with minor variation.
- **AI failure:** Every system diagram becomes boxes, arrows, and clouds with no meaningful distinction.
- **Better move:** Encode different roles differently. Distinguish storage, compute, actors, devices, and outputs.

### 16. Chart truth, narrative falsehood

- **Backrooms symptom:** The room is real; the interpretation is wrong.
- **AI failure:** A chart is numerically accurate but framed to imply something the user should not conclude.
- **Better move:** Align the visual story with the actual claim being made.

## Documents / Markdown / PDFs

### 17. Section inventory instead of information architecture

- **Backrooms symptom:** Segmentation without purpose.
- **AI failure:** A markdown document has Overview, Background, Details, Notes, Thoughts, Misc, Extra, Appendix because documents have sections.
- **Better move:** Organize around decisions, tasks, proofs, and audience needs.

### 18. Context contamination

- **Backrooms symptom:** The room absorbed neighboring reality.
- **AI failure:** A doc written for external readers includes repo-internal names, local shorthand, or nearby task residue.
- **Better move:** Rewrite from the reader's perspective and remove all internal-only terms.

### 19. Repeated explanation

- **Backrooms symptom:** The same corridor returns with slightly different lighting.
- **AI failure:** The same core concept is explained in three sections using different wording.
- **Better move:** Pick the strongest explanation once. Reference it instead of cloning it.

### 20. Nice PDF, bad handout

- **Backrooms symptom:** The environment looks believable until you try to live in it.
- **AI failure:** A handout looks fine as a zoomed PDF but is too dense, too low-contrast, or too small in print.
- **Better move:** Review at actual print size and reading distance.

## Physical Space / Booths / Tables / Signage

### 21. The table is composed like a flat screen

- **Backrooms symptom:** Space exists, but not for bodies.
- **AI failure:** A booth is arranged for screenshot symmetry instead of human movement.
- **Better move:** Think in walk-up order: what people see from afar, at approach, and while standing in front.

### 22. Competing focal points

- **Backrooms symptom:** Multiple corridors suggest themselves, none clearly correct.
- **AI failure:** Prototype, QR code, main monitor, candy, handouts, and logo all compete equally.
- **Better move:** Choose one focal anchor and demote the rest.

### 23. Wrong sightline assumptions

- **Backrooms symptom:** The room assumes a perspective the inhabitant does not have.
- **AI failure:** Important information is placed too low, too high, or too small for standing viewers.
- **Better move:** Design from actual sightlines and reach zones.

### 24. Decorative clutter

- **Backrooms symptom:** Extra objects exist without belonging.
- **AI failure:** Table decor, props, or signage get added because booths should feel full.
- **Better move:** If an object does not support story, usability, or credibility, remove it.

### 25. Screen logic without physical logic

- **Backrooms symptom:** The geometry works only from one impossible angle.
- **AI failure:** A printed sign assumes hover states, interactivity, or close-range reading.
- **Better move:** Match medium to capability.

## Meta Leakage / Context Suction

### 26. Nearby label contamination

- **Backrooms symptom:** Another level bleeds through.
- **AI failure:** The final artifact inherits task labels, sprint names, assignment wording, file names, or prompt-era context.
- **Better move:** Perform a belongs-to-the-user pass on every visible label.

### 27. Tool residue

- **Backrooms symptom:** Strange evidence that the world was generated.
- **AI failure:** Internal toggles, placeholder assets, comments, test values, or export artifacts remain visible.
- **Better move:** Remove evidence of construction unless the audience explicitly needs it.

### 28. Audience drift

- **Backrooms symptom:** The room was built for someone else.
- **AI failure:** A stakeholder deck contains implementation detail for engineers, or product docs read like agent notes.
- **Better move:** Re-anchor on the true audience and rewrite.

## Structure / Narrative

### 29. No primary corridor

- **Backrooms symptom:** Many possible routes, none trustworthy.
- **AI failure:** A page or deck presents multiple equally weighted entry points.
- **Better move:** Choose the primary path and subordinate everything else.

### 30. Over-committed temporary solution

- **Backrooms symptom:** A provisional space became the permanent map.
- **AI failure:** The model builds a rough structure for now, then keeps elaborating it instead of resetting.
- **Better move:** If the skeleton is wrong, restart before adding detail.

### 31. Good local transitions, bad global story

- **Backrooms symptom:** Each turn makes sense until you zoom out.
- **AI failure:** Slide 2 follows slide 1, slide 3 follows slide 2, but the full sequence does not build a persuasive argument.
- **Better move:** Check the whole path, not just adjacent pairs.

## Short Anti-Examples

- Bad: `Assignment 6` on a public site.
  Good: a real product title or no label at all.

- Bad: 90-second booth deck with unreadable 11pt-like text.
  Good: aggressive simplification and room-distance readability.

- Bad: a deck ending around slide 5, then continuing with more mainline business content.
  Good: clear close, then appendix if needed.

- Bad: six decorative pills all saying minor variants of the same thing.
  Good: one strong statement and one supporting proof.

- Bad: putting a QR code, product photo, and CTA in three equally loud places.
  Good: one primary attention anchor, one secondary action.

- Bad: a chart that is visually elegant but does not support the exact claim.
  Good: narrative honesty before aesthetic polish.

## Fast Review Prompt

When in doubt, ask:

- Is this artifact intentional, or does it feel averaged?
- Does anything belong to the process more than the audience?
- Does the artifact already know where it should end?
- Is any important element only working because I am zoomed in or because I know hidden context?
- If a stranger walked into this with no backstory, would it still feel coherent?
